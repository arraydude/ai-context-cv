# ATS (Applicant Tracking Systems) — Best Practices for CV/Resume

Research from the LinkedIn article you shared + additional 2026 sources.

---

## What is ATS?

Software that automates recruitment filtering. **75% of resumes are rejected by ATS before a human sees them.** 97% of Fortune 500 companies use ATS.

### How it works:
1. Company sets position requirements as parameters (keywords, certifications, languages, etc.)
2. ATS scans CVs for matching keywords
3. Resumes are scored and ranked (typically need **70-80% match** to pass)
4. Only top-ranked profiles reach a human recruiter

### "Knockout Questions"
Initial screening questions that pre-filter candidates on non-negotiable requirements (work authorization, language, certifications). If you fail these, your CV is never scored.

## Formatting Rules (Critical)

### Layout
- **Single-column only** — multi-column layouts get scrambled (ATS reads left-to-right, top-to-bottom linearly)
- **Reverse-chronological** order (most recent first)
- Line spacing: 1.0 to 1.15
- Margins: 0.5 to 1.0 inches

### DO NOT USE:
- Tables (content gets scrambled or skipped)
- Text boxes
- Graphics, icons, charts, skill bars, star ratings
- Images/headshots
- Headers/footers for contact info (ATS can't read them)
- Multi-column designs
- Colored backgrounds or light text on dark
- Fancy templates from Canva, etc.
- Hidden white text with keywords (ATS detects this as fraud)

### Safe to use:
- Standard bullet points (• or -)
- Bold text for emphasis
- Simple horizontal lines between sections
- Comma-separated skill lists
- Tab stops for alignment

### Fonts
**Safe:** Arial, Calibri, Helvetica, Roboto, Georgia, Times New Roman, Verdana
- Body: 10-12pt
- Headers: 14-16pt
- Name: 18-24pt

### File Format
- **PDF** (default — preserves formatting, modern ATS parse well)
- **DOCX** if explicitly requested
- **Never:** image-based PDFs, .doc, Google Docs links

### Date Format
- Use **MM/YYYY** (e.g., 03/2022 - 06/2024)
- Never: year only, quarters, seasons, abbreviated years ('23)

## Section Structure

### Required order:
1. **Contact Information** (in body, NOT header/footer)
2. **Professional Summary** (3-5 primary keywords woven in)
3. **Work Experience** (reverse chronological, with metrics)
4. **Skills** (comma-separated text list)
5. **Education**

### Section Headers — use standard names:
| Use this | NOT this |
|----------|----------|
| Contact Information | Reach Me At |
| Professional Summary | About Me |
| Work Experience | Career Journey |
| Skills | Toolbox / Arsenal |
| Education | Academic Background |

## Keyword Strategy

### How to find the right keywords:
1. Read the job description thoroughly
2. Identify: required skills, tools, methodologies, job title variations, soft skills, education requirements
3. Pick **10-15 key terms** from the posting
4. Integrate them naturally — don't stuff

### Where to place keywords:
- **Professional Summary:** 3-5 primary keywords
- **Work Experience:** woven into achievement bullets with metrics
- **Skills section:** listed explicitly as text
- **Education:** relevant coursework if applicable

### Important:
- Include both full terms AND acronyms: "Search Engine Optimization (SEO)"
- Use exact language from job postings
- Modern ATS detects keyword stuffing — keep it natural
- Skills mentioned only in bullet points are less discoverable than ones in a dedicated Skills section — do both

## Writing Achievement Bullets

### Formula: CAR (Context → Action → Result) with metrics

**Bad:** "Worked on performance optimization"
**Good:** "Optimized SQL queries, reducing load times by 40% and saving $50K annually"

**Bad:** "Worked with Python, JavaScript, and AWS"
**Good:** "Developed customer-facing web app using Python and JavaScript, increasing user engagement by 25%"

### Always include:
- Specific technologies used
- Quantifiable results (%, $, time saved, scale)
- Action verbs: Architected, Built, Designed, Led, Migrated, Optimized, Reduced, Scaled

## Testing Your Resume

### Quick test:
1. Open your resume PDF
2. Select all text (Cmd+A)
3. Copy and paste into a plain text editor
4. If the text is readable, in correct order, and contains all info → it will parse well

### Online tools:
- [Jobscan](https://www.jobscan.co/) — compare resume vs job description

## Implications for Our CV

When we build the PDF CV, we must:
- Use single-column, clean layout
- Standard section headers
- No graphics, tables, skill bars, or icons
- Include a dedicated Skills section (comma-separated text)
- Mirror keywords from target job descriptions
- Quantify everything possible
- Use MM/YYYY date format
- Put contact info in body, not header/footer
- Test with Cmd+A → paste into plain text editor
- Generate both PDF and DOCX versions
